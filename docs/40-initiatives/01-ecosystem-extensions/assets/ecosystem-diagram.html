The Copilot Ecosystem: three products, four layers, one command

One resolver, one manifest, one command — parameterized three ways: product × tier × dimension. A person authors at the narrowest layer that fits their audience; content resolves personal → department → org → foundation, nearest wins. A non-technical user runs a single command and gets a working, focus-scoped AI partner — no git, no URLs, no manual wiring.

02

How a machine resolves the stack

One published ecosystem.yml seeds everything. Materialize is a reconciling sync — it prunes what a removed layer or disabled product leaves behind, never orphans it.

derive
Read the seed
Fetch <org>/copilot-ecosystem → compute every repo URL.
pull
Fan-out, 4 layers
Each layer pulled with the right credential from the URL itself.
resolve
Nearest wins
Per name: personal › dept › org › foundation. Shadows reported.
sync
Materialize + prune
Copy winners into ~/.claude/; delete anything no longer resolved.
explain
Provenance
Per item: which layer won, what it shadows, source SHA + signer.
03

Bob’s install — one command, three rings

The chicken-and-egg (the foundation sets itself up) is resolved by a dependency-free bootstrap that installs a capable foundation, which then drives every guided, idempotent, self-healing phase. He has a partner before anything else can fail.

$ /bin/bash -c "$(curl -fsSL https://get.copilot.enac.dev/bootstrap.sh)" # or a double-click offline bundle from IT
Ring 0
bootstrap
Admin-free binaries (no sudo), proxy-aware, offline-capable. Installs prereqs + the foundation + the copilot binary on PATH.
assumes only: a shell
Ring 1
copilot doctor
Idempotent state machine: identity → org → ask dept → dept → personal → materialize → verify → teach. Re-runnable = self-repairing.
install = repair = update
Ring 2
your partner
Memory, Task, protocol, agents — focus-scoped to Bob’s department. Plus copilot add skill --personal so he extends it himself.
two questions, answered in plain English
04

Focus, not corpus

The reason to layer at all: a worker goes deep in their area without carrying the whole organization.

rank 10personalyour notes
rank 20financeyour team’s knowledge
rank 20engineeringnever named · never cloned
rank 30orgcompany base
rank 40foundationpublic engine

A Finance user resolves Finance + org + foundation — and nothing else.

The corpus is bounded at the manifest, before any content loads. Engineering’s department layer is never named, never cloned, never in the index. No behemoth files for content you’ll never use.
